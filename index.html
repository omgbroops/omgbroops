Featured project

Polyshield

Approval, authorization, and audit infrastructure for AI coding agents.

Polyshield is a safety and control layer for AI coding agents and the local MCP tools they use. It acts as a local MCP proxy between an agent and tools like filesystem access, git operations, browser automation, local databases, and shell-like task runners. Every tool call can pass through policies, approvals, prompt-injection defenses, credential isolation, and audit logs before execution.

  • Local MCP proxy for coding agents
  • Approval queue for risky actions
  • Policy layer: allow / deny / require approval
  • Audit trail for every agent action
  • Credential isolation & prompt-injection defenses
  • Filesystem, git, Playwright, local DBs, MCP servers

Built because coding agents are becoming operators, not just autocomplete tools.

Open-source experiment

JSwingX

Roblox-inspired UI patterns for Java Swing.

JSwingX is a Java Swing UI framework I built to bring game-like UI composition patterns into Java desktop interfaces. It uses concepts inspired by Roblox’s UI model, including tree-based composition, scale/offset sizing, anchor-based positioning, reusable UI primitives, tween animations, layout helpers, selectable groups, rich text, and image handling.

  • Tree-based UI composition
  • Dim / Dim2-style sizing
  • Anchor-point positioning
  • Tween animations
  • Layout helpers & interactive components
  • Rich text & image handling

Rice University · Grand Winner

Pumps & Pipes Engineering Competition

Grand Winner — an RC ChemE car combining chemistry, electronics, and robotics.

My team built an RC ChemE car that combined chemistry, electrical engineering, and robotics. The car used an iodine clock reaction as part of its stopping mechanism: when the chemical reaction changed visually, a photoresistor / light-detection circuit detected the change and triggered the control system to stop the motors. I worked on the electronics, programming, motors, sensors, and debugging — the project required heavy iteration, long troubleshooting sessions, and integration across chemistry, hardware, and code.

  • Iodine clock reaction
  • Photoresistor / light detection
  • ESP32 / microcontroller control
  • Motors & sensors
  • Chemistry + robotics + electronics
  • Debugging under time pressure

About

Between creativity and control.

I’m Momo, a builder interested in systems that sit between creativity and control: games, UI frameworks, developer tools, and AI agents. I like projects where design and engineering are both necessary — where the interface matters as much as the architecture behind it.

My path started with indie game development, which made me learn whatever the project needed: scripting, UI design, 3D modeling, animation, graphics, and debugging. That eventually led me into building tools, experimenting with automation, and working on Polyshield, a control layer for coding agents and local MCP tools.

Outside of software, I’m fascinated by mathematics, especially calculus. I like how math gives structure to things that feel continuous, messy, or hard to reason about — and I teach a bit of it on Instagram.
